1.经典：js中的for循环与setTimeout定时器的输出值之间的联系

a.for循环中中使用var

解答：这里按三点作答：

​ 1.var声明的是全局变量，会有变量提升，这里全局只有一个变量i

​ 2.for循环将i循环到了5，所以最后的i就指向了5

​ 3.js是单线程的，定时器是异步执行的

  for (var i = 0; i < 5; i++) {
        setTimeout(function () {
            console.log("定时器" + i);
        })
    }

    //5 * 定时器5

b.for循环中使用let

解答：这里也按三点作答：

      1.let声明的是块级作用域，没有变量提升，每次的变量是相互独立

​ 2.for循环每次循环一次，就声明一次i变量

for (let i = 0; i < 5; i++) {
        setTimeout(function () {
            console.log("定时器" + i);
        })
    }
     //定时器0
     //定时器1
     //定时器2
     //定时器3
     //定时器4

还有一种能让正常输出的方法有：让它变成闭包，内部套一个立即执行函数，将定时器的值作为立即执行函数的返回值

for(var i=0;i<3;i++){
​    (function(i){
​      return setTimeout(()=>{
​        console.log(i)
​      },0)
​    })(i)
  }

强制转换：

Number:能转就转，不能转就是NAN

Boolean：转成true/false,注意undefined、null是false,对象和数组是true

Number NAN 0 1/0 数值/NaN/0 NAN NAN(单元素为数字)
String ‘undefined’ ‘null’ ‘true’/‘false’ 字符串 [Object object] 字符串
Boolean false false 有true/无false 非零true/0、NaN是false true true

隐式转换：

Boolean:在比较运算==、!=、>、<、if、while、算数运算+、-、*、/、%使用boolean值时，遇到下面几种会变成false

​ undefined、null、0、NaN、””

Number：除了+会转换成字符串，其他的运算符都会转换成数值

4.js的==和===

==会产生隐式转换

比较规则如下：两个都是简单类型，转换成数值再进行比较

​ 简单与引用：调用对象的valueOf方法取原始值进行比较

​ 引用与引用：比较是否指向同一个对象

​ null==undefined

​ NAN==NAN返回false

===不会产生隐式转换（类型相同，值也相同）

在比较null和undefined的情况下，用==,其他情况使用===

7.闭包的含义和使用场景

8.什么是作用域、作用域链

作用域分为全局作用域和函数作用域，全局作用域会在页面打开时被创建，在页面关闭时被销毁，全局作用域中有一个全局对象window；函数作用域在函数调用时被创建，在函数执行完毕时被销毁，每调用一次函数作用域就是创建一个新的函数作用域，他们之间时相互独立的，函数作用域可以访问全局作用域，反之不行

作用域链就是在当使用一个变量时，会在当前作用域中找，如果当前作用域没有就会往上一级的作用域找，以此类推，直到找到全局作用域中，如果没有就会报错

10.js中的执行上下文和执行栈

执行上下文是对js代码执行时环境的抽象概念，也就是js代码一旦运行，那么就是运行在执行上下文中

执行上下文分为全局执行上下文：只有一个，函数执行上下文：可以有很多个

执行栈：就是调用栈，用于存储执行上下文【后进先出】

11.事件的代理（事件的委派）

事件的委派利用的是事件的冒泡原理，就是将事件绑定到共同的祖先元素身上，当目标元素被触发时，会通过冒泡机制响应到绑定事件的祖先元素身上，从而触发事件绑定函数

作用：减少事件绑定的次数，提高程序性能

onclick、mousedown、mouseup、keydown、keyup

12.AJAX原理、如何实现

Ajax的原理就是通过xmlhttprequest对象给服务器发送异步请求、服务器返回数据之后，通过js操作DOM更新页面

实现:首先创建一个xmlhttprequest对象：var xhr = new XMLHttpRequest()

​ 然后使用.open方法与服务器建立连接：xhr.open()

​ 然后使用.send方法给服务器发送请求：xhr.send()

​ 然后通过xmlhttprequest提供的onreadystatechange事件监听通信状态：xhr.onreadystatechange()

​ 处理返回的数据，更新至页面中

13.call、apply、bind

相同点：1.都能够改变this的指向 2.第一个参数都是表示this要指向的对象 3.都可以传参

不同点：1.call和bind传入的是参数列表，apply传入的是数组

​ 2.call和apply都是一次性传入参数且立即执行，bind返回的是绑定this之后的函数且不是立即执行
