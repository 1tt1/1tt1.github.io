1.说说你对vue的理解

vue是一个js的框架，也是一个建立单页面应用的web应用框架

vue包含了声明式渲染、组件化系统、客户端路由、状态管理器、数据持久化等，

vue的核心库关心的是视图渲染

并且由于vue的渐进式的特性，让它能够便于和第三方库整合

（渐进式：不强制要求开发者使用某一特定的功能，而是根据需求进行扩展）

2.如何理解vue的双向绑定

vue是MVVM模式的，如果Model中的数据发生变化，那么试图View展示的页面内容也会发生变化，同理，如果用户改变了页面视图中的内容，那么model中的数据也会同样发生变化

双向绑定的流程：

首先需要初始化一个vue实例，通过监听器observer监听属性的变化，一旦属性发生变化，那么就会通知订阅者wacther，每一个watcher会绑定一个更新函数，监测到属性发生变化以后，那么就会调用并执行更新函数，从而更新视图，最后编译器compile会初始化页面中的v-on、v-model等节点绑定的数据，实现双向绑定

3.v-if和v-show区别：

v-show：相当于display:none,只是控制样式的隐藏和显示【不触发vue的生命周期】

v-if：是真实的控制dom节点的增加和删除的【触发beforeCreate Created beforeMount mounted beforeDestory destoryed等声明周期】

【v-if开销更大，如果要频繁切换选择v-show】

5.为什么v-if和v-for不能一起使用

因为v-for的优先级大于v-if

如果在同一元素同时使用两个的话会造成资源浪费，因为每次使用v-if前都要执行一次v-for

解决办法：

在循环出现在条件内部：

<template v-if='isShow'>
	<p v-for='item in items' :key='item.id'></p>
</template>

当条件出现在循环内部：通过computed提前过滤那些不需要显示的项

computed:{
//过滤显示已经展示的元素
	items(){
		return this.list.filter((item)=>{
			return item.isShow
		})
	}
}

9.防抖与节流

防抖：n秒后执行该事件，若在n秒内被触发，则重新计时

节流：在n秒内只执行一次，若在n秒内重复触发，只有一次生效

10.vue实例挂载过程中发生了什么

  • new Vue的时候会调用_init方法
    • 定义 $set、 $get 、$delete、$watch 等方法
    • 定义 $on、$off、$emit等事件
    • 定义 _update、$forceUpdate、$destroy生命周期
  • 调用$mount进行页面的挂载
  • 挂载的时候主要是通过mountComponent方法
  • 定义updateComponent更新函数
  • 执行render生成虚拟DOM
  • _update将虚拟DOM生成真实DOM结构，并且渲染到页面中

11. Vue中给对象添加新属性时，界面不刷新怎么办

  • Vue.set()——添加少量新属性
  • Object.assign()——大量新属性
  • $forcecUpdated()——不知道怎么做，就用强制刷新，但不建议
  • vue3是使用proxy实现数据的响应式的，直接添加新属性就行

12.vue中组件和插件有什么区别

组件注册——组件构成的是功能模块

vue组件注册主要分为全局注册与局部注册

全局注册通过Vue.component方法，第一个参数为组件的名称，第二个参数为传入的配置项

Vue.component('my-component-name', { /* ... */ })

局部注册只需在用到的地方通过components属性注册一个组件

const component1 = {...} // 定义一个组件

export default {
	components:{
		component1   // 局部注册
	}
}

插件注册——插件是对vue的功能补充

插件的注册通过Vue.use()的方式进行注册（安装），第一个参数为插件的名字，第二个参数是可选择的配置项

Vue.use(插件名字,{ /* ... */} )

注意的是：

注册插件的时候，需要在调用 new Vue() 启动应用之前完成

Vue.use会自动阻止多次注册相同插件，只会注册一次

15.vue的mixin

mixin是混入，一般是将多次使用到的相似的代码以对象的形式传入mixins选项中，作为共用的功能代码

混入方式：局部混入和全局混入

局部混入

定义一个mixin对象，有组件options的data、methods属性

1var myMixin = {
2  created: function () {
3    this.hello()
4  },
5  methods: {
6    hello: function () {
7      console.log('hello from mixin!')
8    }
9  }
10}

组件通过mixins属性调用mixin对象

1Vue.component('componentA',{
2  mixins: [myMixin]
3})

该组件在使用的时候，混合了mixin里面的方法，在自动执行create生命钩子，执行hello方法

全局混入

通过Vue.mixin()进行全局的混入

1Vue.mixin({
2  created: function () {
3      console.log("全局混入")
4    }
5})

使用全局混入需要特别注意，因为它会影响到每一个组件实例（包括第三方组件）

PS：全局混入常用于插件的编写

16.vue.observable是什么——可以模拟一个简易的状态管理器vuex【非父子间通信】

Vue.observable，让一个对象变成响应式数据。Vue 内部会用它来处理 data 函数返回的对象

创建一个js文件

1// 引入vue
2import Vue from 'vue
3// 创建state对象，使用observable让state对象可响应
4export let state = Vue.observable({
5  name: '张三',
6  'age': 38
7})
8// 创建对应的方法
9export let mutations = {
10  changeName(name) {
11    state.name = name
12  },
13  setAge(age) {
14    state.age = age
15  }
16}

在.vue文件中直接使用即可

1<template>
2  <div>
3    姓名：{{ name }}
4    年龄：{{ age }}
5    <button @click="changeName('李四')">改变姓名</button>
6    <button @click="setAge(18)">改变年龄</button>
7  </div>
8</template>
9import { state, mutations } from '@/store
10export default {
11  // 在计算属性中拿到值
12  computed: {
13    name() {
14      return state.name
15    },
16    age() {
17      return state.age
18    }
19  },
20  // 调用mutations里面的方法，更新数据
21  methods: {
22    changeName: mutations.changeName,
23    setAge: mutations.setAge
24  }
25}

20.什么是虚拟DOM？如何实现一个虚拟DOM？说说你的思路

虚拟dom是对真实dom的抽象，真实的dom元素体系庞大操作起来非常的消耗性能，虚拟DOM能很好的避免这个问题

如何实现：创建元素节点的时候会生成子元素，每一个子元素也是一个元素节点，这样就形成了一个虚拟树结构，用于描述真实的dom树结构

23.vue怎么实现权限管理

接口权限：登录完成拿到token,将token存起来，通过axios请求拦截器进行拦截，每次请求时头部带token

路由权限：给路由标记相应的权限信息，在每次路由跳转前做校验

菜单权限：路由守卫【根据项目业务确定是否将路由和菜单权限分离】

按钮权限：使用v-if来控制
